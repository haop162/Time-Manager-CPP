## 一、项目概述  【中期必填】

### 1. 项目背景
在快节奏的学习生活中，任务碎片化严重，传统的纸质笔记难以实现快速筛选与实时同步。本项目旨在开发一套轻量级、跨平台的时间管理工具，通过后端逻辑处理与前端看板展示，提升任务处理效率。

### 2. 系统目标
高效录入：通过 C++ 命令行实现任务的快速录入与持久化存储。  

结构化管理：支持任务 ID、时间戳及优先级的自动生成与分类。  

可视化展示：通过 Web 界面实现任务卡片式的实时监控与展示。

### 3. 开发环境
语言：C++ (MinGW g++), HTML5, JavaScript (ES6+), CSS3。  

工具：VS Code, Git。  

通信：基于本地文件系统的异步数据读取（Fetch API）

## 三、系统设计  【中期必填】

### 1. 系统架构
系统采用“生产-消费”模型：C++ 后端负责数据的采集与结构化写入；本地文件系统作为中间件存储；Web 前端负责数据的解析与用户交互展示。
### 2. 模块设计
本项目采用结构化文本存储，每一行数据代表一个对象，字段定义如下：

ID：唯一任务标识（基于 Unix 时间戳）。  

Time：任务创建的格式化时间字符串。  

Priority：任务优先级等级。

## 四、系统实现  【中期部分填写；结项补全】

### 1. 关键技术
文件流操作：利用 std::ofstream 的 app 模式实现非覆盖式增量写入。  

时间戳处理：结合 time_t 与 ctime 获取精准的系统时间，并利用 erase 处理换行符，确保数据行格式规整。

# 1. 用户认证模块 (Identity & Auth)

- **注册功能**: 前端通过 `POST` 请求提交学号、姓名与密码，后端利用 `std::regex` 解析 JSON 数据并与 `users.txt` 进行查重校验，确保学号唯一性。

- **登录与权限控制**: 后端校验用户信息后返回状态码。前端利用 `localStorage` 持久化存储用户信息，并结合 `sessionStorage` 实现了简单的路由守卫（未登录自动踢回首页）。

# 2. 事务日程管理模块 (Task Management)

- **数据持久化**: 实现了事务的“云端保存”。用户添加的任务不再是页面临时展示，而是通过 `/add_task` 接口写入服务器端的 `tasks.txt`。

- **动态拉取渲染**: 页面加载时通过 `window.onload` 自动向后端请求当前用户的专属任务列表，并根据“高/中/低”优先级自动渲染不同的卡片颜色与样式。

# 3. 系统架构与环境

- **后端技术**: 采用轻量级 httplib.h 库，手动处理 CORS 跨域请求，提升了系统的兼容性。

- **数据存储**: 采用轻量级文本文件（`.txt`）存储，配合自定义字符串分割算法，保证了低环境要求下的高读取成功率。

    cors(req, res);
    std::cout << ">>> 正在处理添加请求: " << req.body << std::endl;

    // 分别提取，全部用 m[1]
    std::regex uReg("\"username\":\"(.*?)\""), tReg("\"task_name\":\"(.*?)\""), 
               dReg("\"task_time\":\"(.*?)\""), pReg("\"priority\":\"(.*?)\"");
    std::smatch m;
    std::string u, task, time, prio;
    
    if (std::regex_search(req.body, m, uReg)) u = m[1];
    if (std::regex_search(req.body, m, tReg)) task = m[1];
    if (std::regex_search(req.body, m, dReg)) time = m[1];
    if (std::regex_search(req.body, m, pReg)) prio = m[1];

    if (u.empty() || task.empty()) {
        std::cout << "!!! 错误：解析到的学号或任务名为空" << std::endl;

svr.Get("/get_tasks", [&](const httplib::Request& req, httplib::Response& res) {
    cors(req, res);
    std::string targetUser = trim(req.get_param_value("username"));
    std::string jsonResult = "[";
    
    std::ifstream inFile("tasks.txt");
    std::string line;
    bool first = true;
    while (std::getline(inFile, line)) {
        if (trim(line).empty()) continue;
        
        // 使用更稳健的切割方式
        size_t p1 = line.find(" | ");
        size_t p2 = line.find(" | ", p1 + 3);
        size_t p3 = line.find(" | ", p2 + 3);

        if (p1 != std::string::npos && p2 != std::string::npos && p3 != std::string::npos) {
            std::string u = trim(line.substr(0, p1));
            if (u == targetUser) {
                if (!first) jsonResult += ",";
                jsonResult += "{\"task_name\":\"" + trim(line.substr(p1 + 3, p2 - p1 - 3)) + 
                              "\",\"task_time\":\"" + trim(line.substr(p2 + 3, p3 - p2 - 3)) + 
                              "\",\"priority\":\"" + trim(line.substr(p3 + 3)) + "\"}";
                first = false;
            }
        }
    }
    jsonResult += "]";
    res.set_content(jsonResult, "application/json");
});

当前已完成 C++ 后端核心逻辑开发及前端基础渲染框架的搭建。前后端数据协议已正式打通，实现了任务从终端录入到网页展示的全链路闭环
