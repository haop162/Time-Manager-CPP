## 一、项目概述

### 1. 项目背景
在大学经常有很多任务同时进行，有的优先级高有的低，我们经常会在进行一些比较重要的任务时忘记一些其他任务
所以本项目旨在通过 C++ 实现一套带有智能权值调度算法的工具，帮助用户科学规划

### 2. 项目目标
实现一个轻量级、响应快、具备智能优先权评估功能的日程管理系统，支持任务的动态排序、持久化存储以及 DDL 倒计时提醒，提升用户时间管理效率。
### 3. 开发环境与条件
开发语言： C++ 17

开发工具： Visual Studio 2022

版本管理： Git / GitHub

运行环境： Windows 10/11 控制台

---

## 二、需求分析

### 1. 目标用户与使用场景
目标用户： 需要同时处理多项任务的大学生、开发者。

使用场景： 每日学习计划制定、学期大作业进度追踪、考试周任务紧急程度评估。

### 2. 核心功能清单
任务录入模块： 支持输入任务名、截止日期（年/月/日/时）、任务重要程度权重（1-5级）。智能调度模块： 基于 Priority = Weight/TimeRemaining，公式实时计算并自动更新排序。持久化管理： 自动将任务数据保存至本地磁盘，防止程序关闭后丢失。查询与看板： 实时显示当前最紧迫任务的倒计时。

### 3. 非功能需求
- 性能要求 任务排序算法复杂度需控制在 O(n \log n)以内，确保瞬时响应。
- 安全要求 确保本地文件读写安全，防止因格式错误导致的程序崩溃
- 兼容性要求 完美适配 Windows 控制台字符编码，支持中文输入显示

## 三、技术路线

### 1. 总体架构
采用模块化设计（Modular Design）：

数据层 (Data Layer)： 负责 Task 对象的定义与文件 I/O。

逻辑层 (Logic Layer)： 负责时间差计算、优先队列调度算法。

展示层 (Presentation Layer)： 负责控制台 UI 渲染与用户交互输入。

### 2. 主要技术
前端（交互）： 基于 C++ 标准输入输出（iostream）和格式化控制。

后端（逻辑）： * 使用 <ctime> 处理系统时间。

使用 std::priority_queue 维护任务队列。

使用 std::algorithm 进行多准则排序。

数据库： 基于文件流（fstream）实现的轻量级文本数据库（CSV 或自定义二进制格式）。

### 3. 数据设计初步考虑
主要实体： Task 类（包含 ID, Name, Deadline, Weight, Status）。

关键数据关系：

采用单表结构存储。

核心字段映射关系：TaskID (唯一标识) -> CalculatedPriority (动态计算出的浮点值)。
